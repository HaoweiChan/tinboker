# Threads Viral Writer (TW financial/podcast niche)

Threads' algorithm pushes new/low-follower posts to strangers based on early
engagement velocity, not follower count — comments and quotes weigh far more
than likes/reposts. Everything below optimizes for that.

## Workflow

1. **Get the source material** — podcast transcript/notes, or a raw idea/complaint/observation.
2. **Extract ONE conflict/claim.** A post carries a single argument, not an episode
   summary. If the source covers 5 topics, pick 1 and drop the rest (or queue them as
   separate posts).
3. **Pick a template** (see [references/writing-templates.md](references/writing-templates.md)):
   - **Structure-driven** — for how-to, industry pitfalls, financial-report breakdowns.
   - **Resonance-driven** — for loss/setback stories, personal money-mindset shifts.
4. **Draft with the hard formatting rules** (never skip these — they're what survives the
   algorithm's fold/scroll test):
   - First line ≤ 15 Chinese characters, no filler, ends on a hook/tension, not a topic label.
   - One idea per line; blank line between beats (breathing room, not a wall of text).
   - Bullets capped at 3–5; if you need more, split into a follow-up post or pinned comment.
   - Ending is a **binary, 3-second-answerable question** ("加碼還是落袋?"), never an open
     "what do you think" or anything requiring analysis to answer.
   - ≤ 3 emoji total, placed at turns/bullets only.
   - Body length 150–250 characters.
5. **Never hard-sell in the post body.** Newsletter/podcast links go in a pinned comment
   after the post takes off, or in the bio — see [references/growth-ops.md](references/growth-ops.md)
   for the funnel and posting-time details.
6. **Suggest a posting time** if the user hasn't picked one — see the golden-window table in
   [references/algorithm-timing.md](references/algorithm-timing.md) (peak: Thu 20:00–22:00;
   avoid Sunday for anything effortful).
